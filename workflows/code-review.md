# Workflow: Code Review

Use `templates/code-review-report.md` when a written review artifact is needed.
Pair with `skills/code-review/SKILL.md`, `rules/code-review.md`, and `rules/evidence-and-scope.md`.

1. Read the relevant diff or changed files.
2. Run the findings-first review pass:
   - correctness
   - regressions
   - security and data handling
   - performance and reliability
   - missing tests
3. Report issues with file references and concise rationale.
4. Add a brief summary only after findings.

# Workflow: Project Docs

Use when setting up or updating a lean source-of-truth documentation system for a future project repository.

Pair with `skills/project-docs-system/SKILL.md`, `templates/project-overview.md`, `templates/system-map.md`, and `templates/update-guide.md`.

1. Discover the repository's current documentation, entrypoints, and major directories.
2. Classify each relevant document as:
   - project truth
   - planning or working docs
   - historical or reference docs
3. Decide whether the repository already has a usable source-of-truth system.
4. If a canonical doc already exists for a topic, update it instead of creating a duplicate.
5. If the system is incomplete, create only the missing docs from the lean canonical set:
   - project overview
   - system map or architecture map
   - update guide or doc conventions
6. Put assumptions, open questions, and update guidance inside the canonical docs instead of scattered notes.
7. Keep the resulting doc system lean, repo-appropriate, and easy to maintain.

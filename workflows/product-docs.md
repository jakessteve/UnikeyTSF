# Workflow: Product Docs

Use when creating or updating product-facing planning documents such as PRDs, feature briefs, rollout notes, or implementation summaries.

Pair with `skills/product-docs/SKILL.md` and `templates/feature-spec.md`.

1. Ground in the current product context, code, and existing documentation first.
2. Identify the audience, decision to support, and level of detail needed.
3. Reuse or update an existing product-facing document when there is already a good home.
4. Create a new document only when the information does not fit an existing product-facing artifact.
5. Keep the output concise, decision-oriented, and safe for implementation handoff.
6. Treat product planning docs as supporting artifacts, not canonical source-of-truth project documentation.

# Workflow: Implementation Review

Pair with `skills/testing-verification/SKILL.md`, `rules/delivery-and-verification.md`, and `templates/code-review-report.md` when a reusable review artifact helps.

1. Compare the delivered change to the original plan or request.
2. Check for scope drift and hidden side effects.
3. Verify the claimed proof: tests, build, lint, or reproduction path.
4. Highlight gaps between intended behavior and implemented behavior.
5. End with concrete next actions only if gaps remain.

# Workflow: Plan Feature

1. Ground in the current code and interfaces.
2. State the goal, success criteria, and constraints.
3. Identify the likely behavioral changes and their touch points.
4. Decide whether the work is direct, phased, or delegation-friendly.
5. Choose the artifact shape:
   - use `templates/implementation-plan.md` for implementation and delivery work
   - use `templates/feature-spec.md` for product-facing or design-heavy planning
6. Pair the plan with the relevant skill and rules from `references/task-routing.md`.
7. Define verification before any edits.

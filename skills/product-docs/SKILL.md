# Product Docs

## Focus

- clear outcome
- explicit tradeoffs
- short supporting detail
- implementation-safe wording

## Typical Outputs

- PRD
- feature brief
- implementation plan
- rollout note
- change summary

## Not For

- canonical project overview docs
- system maps that define repo-wide source of truth
- doc-governance or update-guide systems for future projects

Use `skills/project-docs-system/SKILL.md` for source-of-truth project documentation.

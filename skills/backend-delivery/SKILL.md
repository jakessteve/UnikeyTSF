# Backend Delivery

## Workflow

1. Ground in the existing contracts and data flow.
2. Keep interface changes explicit and narrow.
3. Validate inputs, failure modes, and error propagation.
4. Favor deterministic tests where possible.
5. Call out migrations or compatibility implications.

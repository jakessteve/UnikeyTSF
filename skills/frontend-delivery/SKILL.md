# Frontend Delivery

## Workflow

1. Inspect the current UI language and component patterns.
2. Preserve established patterns when the product already has them.
3. For new UI, choose a deliberate visual direction instead of default stacks and generic cards.
4. Keep layout responsive and accessible.
5. Verify behavior on mobile and desktop breakpoints.

## Quality Bar

- clear hierarchy
- purposeful typography
- non-generic spacing and color decisions
- accessible semantics
- minimal but meaningful motion

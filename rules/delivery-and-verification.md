# Rule: Delivery And Verification

- Match the level of process to the task size.
- For small fixes, execute once the relevant code path is understood.
- For medium and large work, use a written plan or template before editing.
- Verify with tests, builds, linters, targeted runs, or careful diff review.
- Do not declare success if verification was skipped or failed.
- If verification cannot run, say exactly what was not checked.
